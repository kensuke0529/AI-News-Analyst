How It Works

Here's a breakdown of the technical architecture I built for this project. It uses RAG (Retrieval-Augmented Generation) with LangGraph to intelligently route queries and deliver accurate, real-time news insights.

I built a Retrieval-Augmented Generation system using ChromaDB. News articles are embedded using OpenAI's text-embedding-3-small model, then retrieved through semantic similarity search to find the relevant context. The system uses LangGraph to create a state machine that decides whether to search recent news or Wikipedia based on the user's query. GPT-4o-mini analyzes the intent and routes it to the right data source.

Through testing and optimization, I improved the relevancy score 40% increase by adjusting chunk sizes from 100 to 500 characters to preserve context. News articles are pulled in real-time from MIT and Techmeme's RSS feed, automatically embedded, and stored in the vector database. The system retrieves the top 3 most relevant chunks for each query, achieving 96% routing accuracy with an average response time of 10.5 seconds.
